# PRD (Product Requirements Document)
# 음식 사진 칼로리 계산 & 다이어트 운동 추천 앱

**버전**: 1.0
**작성일**: 2026-03-26
**작성자**: danggyuLog

---

## 1. 배경 및 목적

### 문제 정의
- 식사 후 칼로리를 직관적으로 파악하기 어려움
- 운동 계획을 세우려 해도 "얼마나 해야 하나"에 대한 기준이 없음
- 기존 칼로리 앱은 음식명을 직접 검색해야 해서 진입 장벽이 높음

### 목적
음식 사진 1장을 업로드하면 AI가 음식을 자동 인식하고,
사용자의 신체 정보 기반으로 칼로리를 분석한 뒤
해당 칼로리를 소모하기 위한 운동 플랜을 즉시 제공한다.

### 사용 흐름
```
사진 업로드 → 신체 정보 입력 → AI 분석 (로딩) → 결과 확인 → 수량 조정 → 재계산
```

## 2. 타겟 사용자

| 유형 | 특징 |
|---|---|
| 다이어트 입문자 | 칼로리 계산이 귀찮아서 포기한 경험 있음 |
| 운동 관심자 | 오늘 먹은 것을 운동으로 소모하고 싶음 |
| 식단 관리자 | 빠른 칼로리 확인이 필요한 사람 |

---

## 3. 핵심 기능 목록

| # | 기능 | 설명 |
|---|---|---|
| F1 | 음식 사진 업로드 | JPG/PNG 드래그앤드롭 또는 파일 선택 버튼 |
| F2 | 음식 인식 (AI) | Claude Vision API로 사진 내 음식을 음식별로 분리 인식. 혼합 음식(예: 김밥)은 재료 합산으로 처리 |
| F3 | 칼로리 분석 | 음식별 칼로리 + 추정량 표시, 총 칼로리 합산 |
| F4 | 사용자 정보 입력 | 체중(kg), 나이, 성별 입력 (BMR 보정에 사용) |
| F5 | 수량 수정 UI | 인식된 음식별 수량을 사용자가 직접 조정 후 재계산 가능 |
| F6 | 운동 소모 계산 | Mifflin-St Jeor BMR + MET 기반으로 나이·성별 보정된 운동별 소모 시간 계산 |
| F7 | 운동 추천 | 유산소 + 근력 혼합 8종, 각 운동별 최소/권장 소모 시간 표시 |
| F8 | 결과 표시 | 음식별 칼로리 카드 / 총 칼로리 / 운동 플랜 카드 형태 출력 |
| F9 | 로딩 상태 표시 | 분석 중 스피너 및 "AI가 식단을 분석 중입니다..." 메시지 표시 |

## 4. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 응답 속도 | 사진 분석 결과 평균 10초 이내 |
| 언어 | 한국어 UI |
| 파일 형식 | JPG, JPEG, PNG만 허용 |
| 파일 크기 | 업로드 이미지 최대 10MB |
| 브라우저 지원 | Chrome, Edge 최신 버전 |
| 보안 | 서버 사이드에서 파일 확장자 및 크기 검증 필수 |

---

## 5. 에러 처리 요구사항

| 상황 | 사용자에게 보여줄 메시지 |
|---|---|
| 지원하지 않는 파일 형식 | "JPG, PNG 파일만 업로드할 수 있습니다." |
| 파일 크기 초과 | "10MB 이하 파일만 업로드할 수 있습니다." |
| 음식 인식 실패 | "음식을 인식하지 못했습니다. 음식이 잘 보이는 사진으로 다시 시도해주세요." |
| API 호출 실패 | "분석 중 오류가 발생했습니다. 잠시 후 다시 시도해주세요." |

---

## 6. 제외 범위 (Out of Scope)

- 회원가입 / 로그인 / 히스토리 저장
- 영양소 상세 분석 (단백질, 지방, 탄수화물)
- 모바일 앱 (웹앱만)
- 운동 강도 선택 옵션 (v2 추가 예정)
- 음식 데이터베이스 직접 관리
